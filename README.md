# Current Affairs Daily

A clean, friendly Android app for daily current-affairs practice, built for Indian
competitive exams (UPSC, SSC, banking, State PSCs).

- **Daily quiz** — 30 MCQs generated every morning. Answer a question and instantly see
  whether you were right, then scroll for the full explanation: the fact, the bigger
  picture / links to past events, and a one-line "why it matters for exams".
- **Weekly test** — 100 MCQs curated from the whole week's news. Exam mode: no feedback
  until you submit, then a full score breakdown and per-question review.
- **Progress** — streaks, accuracy and a history of every attempt (stored on-device).

Questions are generated by **Google Gemini** from curated news RSS feeds, published as
static JSON to this repo by a scheduled GitHub Action, and read by the app over a CDN.
**No server to run, no accounts** (v1 is local-only).

```
current-affairs-app/
├── App.tsx, src/            # the Expo React Native app
├── content/                 # generated question papers (committed by the cron)
│   ├── index.json           #   catalogue the app reads first
│   ├── daily/YYYY-MM-DD.json
│   └── weekly/YYYY-Www.json
├── generator/               # Node scripts: news -> Gemini -> JSON
│   ├── generate-daily.mjs   #   30 questions
│   ├── generate-weekly.mjs  #   100 questions
│   ├── sources.json         #   curated RSS feeds (edit freely)
│   └── lib/
└── .github/workflows/       # daily + weekly cron
```

## 1. Run the app locally

```bash
npm install
npx expo start
```

Press `a` for an Android emulator, or scan the QR code with the **Expo Go** app on your
phone. The app ships with a small sample paper so it works even before the live content
exists.

## 2. Generate questions

Get a Gemini API key at <https://aistudio.google.com/apikey>.

```bash
cd generator
npm install
cp .env.example .env          # paste your key into .env
npm run daily                 # writes content/daily/<today>.json + updates index.json
npm run weekly                # writes content/weekly/<week>.json
```

Tune the feeds in [`generator/sources.json`](generator/sources.json) — any feed that
fails is skipped automatically. Models default to `gemini-2.5-flash` (daily) and
`gemini-2.5-pro` (weekly); override with `GEMINI_MODEL`.

## 3. Automate it (GitHub Actions)

Once this repo is on GitHub:

1. **Add the API key as a secret** — repo → *Settings* → *Secrets and variables* →
   *Actions* → *New repository secret*: name `GEMINI_API_KEY`, value = your key.
2. (Optional) override models via *Variables*: `DAILY_MODEL`, `WEEKLY_MODEL`.
3. The workflows run on schedule (daily 07:00 IST, weekly Sunday 07:30 IST) and commit
   fresh papers back to `content/`. You can also trigger them manually from the
   **Actions** tab (*Run workflow*).

The app fetches `content/index.json` from `raw.githubusercontent.com` (with a jsDelivr
CDN fallback), so new papers appear with no app update.

## 4. Build the downloadable APK

The app uses [EAS Build](https://docs.expo.dev/build/introduction/) — no Android Studio
needed.

```bash
npm install -g eas-cli
eas login                     # create a free account at https://expo.dev if needed
eas build:configure           # links this project (writes the projectId)
eas build -p android --profile preview
```

When the build finishes, EAS prints a **public URL** with a **Download** button for the
`.apk` — share that link and anyone can install it (Android → allow "install from unknown
sources"). Rebuild anytime to get a new link; bump `versionCode` in `app.json` for updates.

> iOS is planned for a later release. The codebase is already cross-platform; it only
> needs an Apple Developer account and an `ios` EAS profile when you're ready.

## How question quality is enforced

The generator prompt ([`generator/lib/prompt.mjs`](generator/lib/prompt.mjs)) instructs
Gemini to ground every question strictly in the supplied news, cover the exact angles
exams test (appointments, awards, dates, sports, schemes, economy, defence, S&T, etc.),
write same-type distractors, spread the correct option across A–D, and always include the
explanation, the bigger-picture background, and an exam-relevance line. Output is
constrained to a strict JSON schema and then validated and de-duplicated in code before
publishing.
